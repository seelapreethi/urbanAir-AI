# Coding Guidelines

## General

- Follow Clean Architecture
- Modular code
- Reusable components
- No duplicated code
- Strong typing
- Production-ready code

---

## React

- Functional Components
- Hooks only
- TypeScript
- Server Components where appropriate
- Client Components only when needed

---

## Styling

- TailwindCSS
- ShadCN UI
- Mobile First
- Dark Mode
- Accessible UI

---

## Backend

- FastAPI
- Dependency Injection
- Service Layer
- Repository Layer
- Pydantic Validation
- SQLAlchemy ORM

---

## AI

- Modular Pipelines
- Configurable Models
- Explainable Predictions
- Free Models Only

---

## Git

Commit Format

feat:

fix:

refactor:

docs:

style:

test:

---

## Naming

Components

PascalCase

Functions

camelCase

Variables

camelCase

Constants

UPPER_SNAKE_CASE

Files

kebab-case

---

## Documentation

Every public function should include a docstring or clear comments where appropriate.

---

## Performance

- Lazy Loading
- Dynamic Imports
- Memoization where useful
- Pagination
- Caching

---

## Security

- Validate inputs
- JWT Authentication
- Environment variables
- Never expose secrets

---

## Rule

Never generate multiple unrelated files.

Generate one logical feature at a time.

Always follow the architecture documents.
